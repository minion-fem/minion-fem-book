# Neural Topology Optimisation

Topology optimisation (TO) finds the material layout inside a design domain that minimises structural compliance (maximises stiffness) subject to a volume constraint. This section covers the **neural reparametrisation** approach: the density field is represented by a neural network rather than a grid of independent variables.

---

## Why neural reparametrisation?

The classical approach, **SIMP** (Solid Isotropic Material with Penalization), assigns an independent density variable $\rho_e \in [0,1]$ to each element and penalises intermediate densities via $E_e = E_{\min} + \rho_e^p (E - E_{\min})$. This works well on structured grids but has two drawbacks:

1. **Density filtering is mesh-dependent.** Minimum length scale is enforced through spatial filters that depend on the mesh topology.
2. **The design space is $\mathbb{R}^{n_{\text{elem}}}$.** For large 3D models with millions of elements this is expensive, and the independent-variable parameterisation carries no geometric prior.

**Neural reparametrisation** replaces the density vector with a neural network $\rho_e = \mathcal{N}_\theta(\mathbf{x}_e)$, where $\mathbf{x}_e$ is the centroid coordinate of element $e$ and $\theta$ are the network weights. The optimisation variable is now $\theta$, and the design is generated by a forward pass:

$$\rho_e = \mathcal{N}_\theta(\mathbf{x}_e) \in (0, 1)$$

This has two key advantages:

- **Implicit regularisation.** Network weights encode a smooth, band-limited function; high-frequency checkerboard patterns are naturally suppressed.
- **Mesh-independence.** The coordinate input works on any mesh — structured or unstructured — with no changes to the filter.

---

## Compliance minimisation problem

The structural optimisation problem is:

$$\min_\theta \quad C(\theta) = \mathbf{f}^T \mathbf{u}(\theta)$$

$$\text{subject to} \quad \frac{\sum_e \rho_e(\theta) V_e}{V_0} \leq v_f, \quad \mathbf{K}(\theta)\,\mathbf{u} = \mathbf{f}$$

where $C$ is the structural compliance, $\mathbf{K}$ is the global stiffness matrix assembled with SIMP-penalised moduli, $v_f$ is the target volume fraction, and $V_e$ is the element volume.

### SIMP penalisation

Element stiffness is scaled by the density via:

$$E_e(\rho_e) = E_{\min} + \rho_e^p\,(E - E_{\min})$$

where $p \geq 1$ is the penalisation exponent. High $p$ drives elements to 0–1 (void–solid) by making intermediate densities energetically expensive. Minion uses **penalty continuation**: $p$ starts at 1 (no penalty) and increases by 0.05 per iteration up to a maximum of 4, gradually sharpening the 0–1 transition as the optimisation progresses.

### Volume constraint

The volume constraint is enforced either via a **Lagrangian penalty**:

$$\mathcal{L} = \frac{C(\theta)}{C_0} + \alpha \left(\frac{\bar{\rho}}{v_f} - 1\right)^2$$

with $\alpha$ growing from 0.2 each iteration, or via a **hard constraint** that shifts the sigmoid so that its mean is exactly $v_f$ at every step (`hard_volconstraint` flag).

## Training pipeline

The training loop alternates between two phases:

```
for each iteration:
    1. Forward pass: ρ_e = N_θ(x_e) for all elements
    2. FEM solve:    K(ρ) u = f  →  element compliances c_e
    3. Compliance:   C = Σ E(ρ_e) c_e   [differentiable in ρ]
    4. Loss:         L = C/C₀ + α(v̄/vf − 1)²
    5. Backward:     ∂L/∂θ via autograd through ρ_e → L
    6. Update:       θ ← θ − lr · ∇_θ L    (Adam + AMSGRAD)
```

The key design choice in step 3 is that gradients **do not pass through the FEM solve**. Element compliance values $c_e$ are extracted from the FEM result and treated as constants; PyTorch autograd differentiates only through the SIMP scaling $E(\rho_e)$. This is exact for compliance minimisation because:

$$\frac{\partial C}{\partial \rho_e} = p\,\rho_e^{p-1}(E - E_{\min})\,c_e$$

and $c_e = \mathbf{u}_e^T \mathbf{k}_0 \mathbf{u}_e / E$ is fixed once the FEM solve is done.

### Convergence

Training stops when all of the following hold for more than 10 consecutive iterations:

- Grey element fraction $< 3.5\%$ (fraction of elements with $0.2 < \rho < 0.8$)
- Relative compliance change $< 0.5\%$
- Volume fraction error $< 0.5\%$ of target

---

## Interface to the FEM solver

The training loop interfaces with the FEM solver via a thin wrapper (`MinionToFemInterface`). Each iteration it:

1. Sets the density field (`ax_fem.density_field = ρ`)
2. Calls a lightweight single-iteration FEM solve (`solve_for_to()`)
3. Extracts unscaled element compliances $c_e = \mathbf{u}_e^T \mathbf{k}_0 \mathbf{u}_e / E$

For large 3D problems, a PETSc CG + GAMG iterative solver can replace the default direct solver — each TO iteration becomes a single preconditioned CG solve rather than a factorisation.
